# Planning

> If a dedicated planning skill set is installed (e.g. brainstorming or
> writing-plans), prefer it for open design questions or plans someone else will
> execute. This section is for quick in-session decomposition when you're about
> to do the work yourself, right now.

Model: `fast` (see `../model-registry.json`). Decompose the task before doing any work.

1. Identify what needs to happen (not how yet)
2. Break into independent vs. sequential subtasks
3. Flag which subtasks can be delegated (see `sub-agents.md`)
4. Identify what context is actually needed — load nothing yet
5. Switch to the model mapped for the execution task type

Don't research the codebase during planning. Plan first, then load only what execution requires.
